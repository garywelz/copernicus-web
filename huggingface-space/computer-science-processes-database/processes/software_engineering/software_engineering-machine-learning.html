Agile Scrum Sprint

Computer Science Software Engineering Complexity: high
← Back to Computer Science Database Programming Framework

Description

A Scrum sprint delivers a potentially shippable increment in a fixed timebox: the team plans and selects a sprint backlog from the product backlog, develops through daily standups, removing impediments as they arise, and when the timebox ends holds a sprint review to demo and a retrospective to improve.
