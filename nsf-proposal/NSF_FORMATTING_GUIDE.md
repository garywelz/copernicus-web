# NSF FastLane/Research.gov Formatting Guide

## NSF Formatting Requirements

### General Requirements
- **Font:** 11pt or larger, readable font (Times New Roman, Arial, Helvetica, or similar)
- **Margins:** Minimum 1 inch on all sides
- **Line Spacing:** Single or double spacing acceptable
- **Page Numbers:** Include page numbers
- **File Format:** PDF for upload to FastLane/Research.gov

### Page Limits

1. **Project Summary:** 1 page maximum
2. **Project Description:** 15 pages maximum (including all figures, tables, and references)
3. **References Cited:** No page limit (but should be reasonable)
4. **Biographical Sketch:** 2 pages maximum per person
5. **Current and Pending Support:** No page limit (but should be concise)
6. **Budget Justification:** No page limit (but should be reasonable)
7. **Facilities, Equipment, and Other Resources:** No page limit (but should be concise)
8. **Data Management and Sharing Plan:** 2 pages maximum

### Section Requirements

#### Project Summary (1 page)
- Must include: Overview, Intellectual Merit, Broader Impacts
- Single page, no exceptions
- Can be uploaded as separate document or included in Project Description

#### Project Description (15 pages max)
- Must include:
  - Introduction and Motivation
  - Research Objectives
  - Technical Approach
  - Research Methods and Evaluation
  - Expected Outcomes
  - Broader Impacts
  - Project Management and Timeline
- Can include figures, tables, references within the 15-page limit
- References can be listed within Project Description or separately

#### References Cited
- Can be included in Project Description (within 15 pages) or as separate document
- Standard academic citation format
- No page limit if separate, but should be reasonable

### File Upload Structure

**FastLane/Research.gov typically requires:**
1. Cover Sheet (filled in online)
2. Project Summary (PDF, 1 page)
3. Project Description (PDF, 15 pages max)
4. References Cited (PDF, if separate from Project Description)
5. Biographical Sketch (PDF, 2 pages max)
6. Current and Pending Support (PDF)
7. Budget (entered online or uploaded)
8. Budget Justification (PDF)
9. Facilities, Equipment, and Other Resources (PDF)
10. Data Management and Sharing Plan (PDF, 2 pages max)

### Formatting Checklist

- [ ] Font size: 11pt or larger
- [ ] Margins: 1 inch minimum
- [ ] Page numbers included
- [ ] Project Summary: Exactly 1 page
- [ ] Project Description: 15 pages or less
- [ ] All figures/tables properly formatted
- [ ] References properly formatted
- [ ] No prohibited content
- [ ] All URLs functional (check before submission)
- [ ] Consistent formatting throughout

### Prohibited Content

- Do NOT include:
  - Social Security Numbers
  - Personal information of others without consent
  - Proprietary or confidential information (unless necessary and properly marked)
  - URLs to non-public websites
  - Links to copyrighted material without permission

### Tips

1. **Page Count:** Count carefully - NSF is strict about page limits
2. **Figures/Tables:** Include within Project Description page limit
3. **References:** Can be brief - focus on most relevant citations
4. **Consistency:** Use consistent formatting throughout
5. **PDF Quality:** Ensure PDFs are readable and not corrupted
6. **File Names:** Use clear, descriptive file names
